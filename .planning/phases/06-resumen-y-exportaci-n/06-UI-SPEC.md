# Phase 6 — UI Design Contract

> Visual and interaction contract for the **Resumen y Exportación** screen (RESUMEN-01/02/03, EXPORT-01/02).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is a closing screen, not a new design system.** Every token, button style, and color below
> is already in production across Phases 3–5. Reuse, do not reinvent. Source of truth for tokens:
> `tailwind.config.js`. Source of truth for component patterns: `Landing.jsx`,
> `ConfiguratorWizard.jsx`, `BarraPrecio.jsx`, `PasoExtras.jsx`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-rolled Tailwind v3 — `tailwind.config.js`) |
| Preset | not applicable — shadcn deliberately NOT initialized; sober/industrial identity is bespoke and locked in PROJECT.md/CLAUDE.md. Initializing shadcn would conflict with the established palette and 5 phases of shipped UI. |
| Component library | none (native JSX + Tailwind utility classes; SVG plano is native) |
| Icon library | none — inline SVG only (matches `PasoUso.jsx` pattern). No icon dependency. For the two export buttons, use inline `<svg>` glyphs (WhatsApp mark, download arrow) authored as small inline paths, consistent with the project's "no icon library" stance. |
| Font | Inter (`font-sans`), with `system-ui, -apple-system, Segoe UI, Roboto, sans-serif` fallbacks. Already loaded; do not add a font dependency. |

**Identity reminder (locked):** sober / industrial, agricultural-machinery catalog feel — NOT startup tech.
No gradients, no glassmorphism, no drop shadows beyond the single subtle one already used on the mobile
sticky bar. Square-ish `rounded` corners (Tailwind default `rounded`, not `rounded-xl`). Trato de usted
in 100% of copy. Argentine `$` formatting via `formatPrecio` (never hand-format a number).

## Spacing Scale

Declared values (all Tailwind v3 defaults already in use across the codebase — multiples of 4):

| Token | Value | Tailwind | Usage in Phase 6 |
|-------|-------|----------|------------------|
| xs | 4px | `gap-1` / `space-y-1` | Gaps between budget sub-lines (neto/IVA), inline label-value rows |
| sm | 8px | `gap-2` / `p-2` / `mt-2` | Spacing inside a budget line item, chip padding |
| md | 16px | `p-4` / `gap-4` / `mt-4` | Default card padding (matches `BarraPrecio` `p-4`), gap between config sections |
| lg | 24px | `p-6` / `mt-6` / `gap-6` | Outer page padding (matches wizard `main` `p-6`), space between major resumen blocks |
| xl | 32px | `mt-8` / `gap-8` | Separation between the plano block, the budget block, and the export-actions block |
| 2xl | 48px | `py-12` | Top/bottom breathing room on the full-width resumen container (matches Landing `py-12`) |
| 3xl | 64px | `mt-16` | Reserved — final footer/contact separation if needed |

**Exceptions:**
- **Touch targets: `min-h-[44px]`** on every interactive element (buttons, "Editar" links, export
  actions). This is the existing project standard (Landing, Wizard nav, PasoExtras labels). 44px is a
  deliberate exception to the 4-grid for tap ergonomics on the ~375px Samsung target — keep it.

## Typography

Reuse the exact ramp already shipped (Landing `h1` is `text-3xl`, steps use `text-xl`, body is
`text-sm`/`text-base`, fine print is `text-xs`). Exactly 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Tailwind | Where in Phase 6 |
|------|------|--------|-------------|----------|------------------|
| Display | 28px (`text-3xl` ≈ 30px, leading-tight) | semibold/bold (600/700) | 1.2 (`leading-tight`) | `text-3xl font-bold leading-tight` | Resumen screen title ("Su casilla, lista para enviar" / equivalent) |
| Heading | 20px (`text-xl`) | semibold (600) | 1.2 | `text-xl font-semibold` | Section headers ("Modelo", "Baño", budget "Presupuesto estimado", "Financiación") |
| Body | 16px (`text-base`) / 14px (`text-sm`) | regular (400) | 1.5 | `text-base` / `text-sm` | Config values, budget line labels, financing detail text |
| Label / fine print | 12px (`text-xs`) | regular (400) / semibold for emphasis | 1.5 | `text-xs text-impacar-texto/70` | "+ IVA" suffixes, "orientativo, sujeto a confirmación" note, contact footer, subgroup tags |

**Weights: exactly two — regular (400) and semibold (600).** Bold (700) is permitted ONLY for the
"IMPACAR" wordmark (matches Landing/header). Totals and section headers use semibold, never bold.

**Line heights:** body 1.5 (Tailwind default), headings/display `leading-tight` (1.2). Do not introduce
new line-height values.

## Color

60/30/10 split is already enforced by the palette. The resumen inherits it unchanged.

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#F5F5F0` | `bg-impacar-fondo` | Page background, the full-width resumen surface |
| Secondary (30%) | `#FFFFFF` at 40% (`bg-white/40`) + card borders `border-impacar-texto/10` | `bg-white/40 border border-impacar-texto/10 rounded` | Config section cards, budget card, financing card (matches the wizard `section` card and `BarraPrecio` container) |
| Accent (10%) | `#2D5016` verde campo | `bg-impacar-campo` / `text-impacar-campo` | See reserved-for list below |
| Secondary accent | `#8B6914` cobre | `text-impacar-cobre` | Plano dimension lines (existing) + hover state of the "Volver a empezar" text link only. Do NOT use cobre for primary actions. |
| Destructive | none | — | **No destructive actions in this phase.** No delete, no irreversible operation on this screen. "Volver a empezar" lives in the wizard, not the resumen. |

**Accent (`impacar-campo`) reserved for — explicit list:**
1. The primary CTA section headers/emphasis already established (section titles use `text-impacar-campo` for the small subgroup labels, matching `PasoExtras`).
2. The **WhatsApp export button** (filled `bg-impacar-campo`, white text) — the primary action of the screen.
3. The **budget Total c/IVA line** (`font-semibold text-impacar-campo`) — exactly as `BarraPrecio` already renders it.
4. The "IMPACAR" wordmark and any selected/active state.
5. Focus rings (`focus:ring-2 focus:ring-impacar-campo/40`).

**Accent is NOT used for:** body text, every interactive element indiscriminately, the PDF button fill
(it is an outline button), or decorative backgrounds.

**Plano zone colors** (`zona-bano #BFE3EE`, `zona-dormitorio #C9A66B`, `zona-cocina #A7C796`) are owned
by `FloorPlan.jsx` and reused verbatim in both the on-screen resumen plano and the vector PDF plano. Do
not restyle them.

**WhatsApp button color decision (locked per CONTEXT phase_context):** use the project's own
`impacar-campo` green for the WhatsApp button rather than the external WhatsApp brand green (#25D366) —
this keeps the sober/industrial identity coherent and avoids a foreign brand color. The button is made
recognizable by its inline WhatsApp glyph + label "Enviar por WhatsApp", not by brand color. The PDF
button is the **secondary/outline** variant (`border-impacar-campo text-impacar-campo`, transparent
fill) — same outline style as the wizard "Anterior" button.

## Phase 6 Surface Specs (component-level contract)

These are the NEW surfaces. Each must compose only from the tokens above.

### S1 — Resumen screen shell (`vista === 'resumen'`)
- Full-width, NO progress bar, NO sticky side plano (CONTEXT D-01). Reuses the page chrome:
  `min-h-screen bg-impacar-fondo text-impacar-texto font-sans` + the same `bg-impacar-campo` header
  band with the "IMPACAR" wordmark used in `ConfiguratorWizard.jsx`.
- Content container: `mx-auto max-w-3xl px-6 py-12` (narrower than the wizard's `max-w-5xl` because
  it's a single readable column, not a two-pane layout).
- Vertical rhythm: Display title → subtitle → plano block → config sections → budget block → financing
  block → export actions → global "Volver a editar". Blocks separated by `mt-8` (xl).
- Mobile-first at ~375px: everything stacks in one column with no horizontal scroll. The plano scales
  to container width (SVG `width:100%`, preserves aspect ratio). No layout change needed between mobile
  and desktop beyond the `max-w-3xl` centering — single-column at every breakpoint.

### S2 — Final plano block
- Reuses `FloorPlan.jsx` verbatim with `configDesdeEstado(estado)`. Wrap in the standard card:
  `rounded border border-impacar-texto/10 bg-white/40 p-4`. No new SVG. Same component feeds the PDF
  vector embed (svg2pdf).

### S3 — Config sections with per-section "Editar" (D-02, D-07)
- One card per step group: `Uso y ocupantes / Modelo / Baño / Dormitorio / Cocina y estar / Extras`.
- Card: `rounded border border-impacar-texto/10 bg-white/40 p-4`, `space-y-2` between rows.
- Header row: `text-xl font-semibold` section title on the left, an `Editar` link on the right
  (`ml-auto`), styled as a text link in `text-sm font-medium text-impacar-campo` with
  `min-h-[44px]` hit area, `underline-offset-2 hover:underline focus:ring-2 focus:ring-impacar-campo/40`.
  Clicking dispatches `IR_A_PASO` to that step's index AND switches `vista` back to `'wizard'`.
- Values: ids translated to real names (model `Nx` + largo, baño estándar/ampliado, camas C/S/M with
  counts, horno/heladera/mesa, confort/energía extras). Data-driven from `/data` — never hardcode a
  label. Use `text-sm` body; emphasize the chosen value, mute helper text at `text-impacar-texto/70`.
- A section with no selection renders `Sin selección` (never blank).

### S4 — Itemized budget block (D-03, D-04, D-05) — "expanded BarraPrecio"
- Card header `Presupuesto estimado` (`text-xl font-semibold` — note: BarraPrecio uses `text-xs` for
  its compact header, but the resumen is the full presentation, so promote it to a real heading).
- **Line items**, fed by the new `detallePresupuesto(estado)` helper (`{ items, neto, iva, total }`):
  1. Model base line: `{nombre modelo}` … `{formatPrecio(precioNeto)}` (right-aligned, `flex justify-between`).
  2. Accessory line items **grouped by category** with a small `text-xs font-semibold text-impacar-campo`
     subheader per group (`Baño`, `Dormitorio`, `Cocina y estar`, `Confort`, `Energía`). Each item:
     `text-sm` name left, `text-sm` formatted price right. Indent group items with `pl-3`.
  3. A divider (`border-t border-impacar-texto/10`), then the `Neto` / `IVA 21%` / `Total c/IVA` rows —
     identical markup and styling to `BarraPrecio` (`Total c/IVA` is `font-semibold text-impacar-campo`).
- Orientative note directly under the total: `text-xs text-impacar-texto/70` —
  `Presupuesto orientativo, sujeto a confirmación del asesor comercial.`
- Empty group categories are omitted (don't render an empty "Dormitorio" subheader).
- Never `$NaN`: all numbers through `formatPrecio`; corrupt state → 0, handled upstream by the motor.

### S5 — Financing block (D-06, RESUMEN-03)
- Card with header `Opciones de financiación` (`text-xl font-semibold`).
- `FINANCIACION.map` → each option: `nombre` in `text-sm font-semibold`, `detalle` in
  `text-sm text-impacar-texto/70` below. `space-y-3` between options. Text only, no amounts.

### S6 — Export actions block (EXPORT-01, EXPORT-02)
- Two buttons, stacked on mobile (`flex flex-col gap-3`), side by side on `sm:` (`sm:flex-row`).
  Both `min-h-[44px]`, `rounded`, `font-semibold`, `focus:ring-2 focus:ring-impacar-campo/40`.
- **Primary — WhatsApp** (`bg-impacar-campo text-white hover:bg-impacar-campo/90`): inline WhatsApp
  glyph + `Enviar por WhatsApp`. Renders as an `<a href="https://wa.me/{numero}?text={encoded}">` with
  `target="_blank" rel="noopener noreferrer"`. Number from `src/data/contacto.js`.
- **Secondary — PDF** (outline: `border border-impacar-campo text-impacar-campo bg-transparent
  hover:bg-impacar-campo/5`): inline download-arrow glyph + `Descargar PDF`. `<button type="button">`
  that triggers the jsPDF/svg2pdf generation. While generating, show disabled state
  (`disabled:opacity-40 disabled:cursor-not-allowed`) — reuse the existing disabled pattern.
- Below the buttons, the global text link `Volver a editar` (`text-sm font-medium
  text-impacar-texto/60 hover:text-impacar-cobre hover:underline`, `min-h-[44px]`) — same treatment as
  the wizard "Volver a empezar" link. Returns to `vista='wizard'` without resetting state.

## Accessibility (baseline only — full audit is Phase 7)

This phase ships sane defaults; the end-to-end keyboard/contrast audit is deferred to Phase 7 (UX-02).
- Every interactive element is a real `<button>` or `<a>` with a visible focus ring (existing
  `focus:ring-2` pattern) and `min-h-[44px]` hit area.
- The plano `<svg>` keeps its existing `role="img"` + `<title>` (from FloorPlan).
- Color is never the only signal: the WhatsApp/PDF buttons carry text labels + glyphs, not color alone.
- Contrast: `impacar-campo (#2D5016)` on white and `impacar-texto (#1A1A1A)` on `#F5F5F0` already pass
  AA at body sizes (verified in prior phases). The `text-impacar-texto/70` muted tone is for
  non-essential helper text only, never for the budget total or section values.
