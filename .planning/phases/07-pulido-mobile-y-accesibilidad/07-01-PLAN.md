Patron de foco (RESEARCH Pattern 1, ya citado canonico). El proyecto hoy tiene 0 usos de
.focus()/tabIndex/autoFocus en src/ (grep verificado):
```jsx
import { useRef, useEffect } from 'react'
const headingRef = useRef(null)
useEffect(() => { headingRef.current?.focus() }, [/* dependencia: vista o pasoActual */])
// en el JSX del heading destino:
<h1 ref={headingRef} tabIndex={-1} className="... focus:outline-none">...</h1>
```
Notas: tabIndex={-1} hace el heading focuseable por JS pero NO lo mete en el orden natural de Tab.
Anadir focus:outline-none al heading para no mostrar un contorno inesperado en un titulo.

App.jsx (estructura actual — ruteo de estado SPA, sin router):
```jsx
export default function App() {
  const [vista, setVista] = useState(vistaInicial)   // 'landing' | 'wizard' | 'resumen'
  const { estado, dispatch, reiniciar } = usePersistedConfig()
  return (
    <div className="min-h-screen bg-impacar-fondo text-impacar-texto font-sans">
      {vista === 'landing' && <Landing onComenzar={() => setVista('wizard')} />}
      {vista === 'wizard' && <ConfiguratorWizard ... onVolverInicio={() => setVista('landing')} onVerResumen={() => setVista('resumen')} />}
      {vista === 'resumen' && <Resumen ... onVolverEditar={() => setVista('wizard')} />}
    </div>
  )
}
```
IMPORTANTE: el heading visible de cada vista vive DENTRO de su componente hijo (Landing tiene <h1>,
ConfiguratorWizard tiene <h1>IMPACAR en su <header>, Resumen tiene <h1>IMPACAR). Como App no renderiza
esos headings directamente, el foco al cambiar de vista se gestiona desde App con un contenedor
focuseable por vista (un wrapper con ref + tabIndex={-1}) que recibe el foco al montar cada vista.
