<objective>
Adjuntar el PDF del resumen al envío por WhatsApp usando la Web Share API (con archivo), con
fallback al comportamiento actual (descarga del PDF + link wa.me con texto pre-armado) en entornos
sin soporte.

Purpose: hoy el cliente recibe el texto por wa.me pero el PDF va "aparte" (descarga manual + adjuntar
a mano). Con Web Share API el PDF viaja adjunto en un solo gesto en móvil (el target Samsung ~6"),
reduciendo fricción. El fallback preserva el flujo actual donde la API no está disponible.

Output: exportPDF refactorizado (doc reutilizable + Blob/File), nueva util compartirPorWhatsApp con
detección de soporte, mensajeWhatsApp parametrizado, y el botón de WhatsApp convertido en flujo async.

Nota de plataforma: la Web Share API con `files` SOLO funciona en contexto seguro (HTTPS/localhost)
y en navegadores soportados (Chrome/Edge Android, Safari iOS/macOS; Firefox desktop NO). El share
sheet del sistema no puede pre-seleccionar WhatsApp (limitación de la API — el usuario lo elige en
la hoja). El fallback cubre todos los demás entornos: no hay camino sin salida.
</objective>
