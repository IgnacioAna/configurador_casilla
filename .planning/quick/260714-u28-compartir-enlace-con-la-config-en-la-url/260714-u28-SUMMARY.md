# Quick Task 260714-u28: Compartir enlace con la config en la URL — Summary

**One-liner:** El envío por WhatsApp ahora comparte un ENLACE que abre el configurador con la casilla del cliente ya cargada (plano vivo + presupuesto), en vez de un PDF/texto plano — con tarjeta de vista previa (OG tags) para que el link se vea profesional.

## Contexto (por qué)

Iteración sobre quick-260714-pg2. El flujo previo (wa.me + descarga de PDF) dejaba un texto plano que
prometía un PDF que el cliente no adjuntaba → "la magia del plano se perdía". Decisión del usuario:
compartir el **configurador vivo** vía un link con la config adentro. Es más magia que un PDF (el
asesor ve la casilla viva y puede tocarla) y encaja con el core value del proyecto.

## Qué se hizo

### Task 1 — configLink.js (codificar/decodificar la config en la URL)
- `codificarConfig(estado)` → token compacto versionado (~18 chars). `decodificarConfig(token)` →
  estado completo mergeado sobre estadoInicial, o null si invalido/incompatible. `enlaceConfigApp`
  arma la URL absoluta con `window.location.origin` (sin hardcodear el dominio). 9 tests (round-trip,
  robustez ante tokens adulterados, bitmask fuera de rango).

### Task 2 — Cablear la lectura de la URL
- `usePersistedConfig`: `estadoDesdeURL()` con precedencia sobre localStorage; `limpiarTokenURL()`
  borra el `?c=` tras sembrar (F5 posterior usa localStorage). `App`: link compartido abre directo
  en 'resumen'. `mensajeWhatsApp` reemplaza la linea del PDF por "Vea el plano ... aca:" + enlace.
  `AccionesExport`: boton WhatsApp = `<a>` simple (se quito la descarga automatica de PDF y su nota).

### Task 3 — OG tags + tarjeta estatica
- `index.html`: title/description + Open Graph + Twitter Card. `public/og.png` (1200x630, croquis de
  casilla con la paleta Impacar), generado con canvas y verificado como PNG valido.
